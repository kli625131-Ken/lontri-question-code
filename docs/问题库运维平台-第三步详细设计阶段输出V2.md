## 一、API 接口设计文档

### 1.1 接口规范

- 基础路径：/api/v1
- 认证方式：JWT Token
- 数据格式：application/json
- 字符编码：UTF-8

**通用响应格式：**
```json
{"code": 200, "message": "success", "data": {}}
```

**错误码：**
| code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 / Token 失效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

### 1.3 核心接口详情

**P01 GET /problems — 问题列表查询**

请求参数：project / status / device_type / handler / start_date / end_date / keyword / page / page_size

响应：
```json
{
  "code": 200,
  "data": {
    "total": 128,
    "page": 1,
    "page_size": 20,
    "items": [{
      "id": 1,
      "problem_no": "ABBP6-202603-001",
      "project_code": "ABBP6",
      "device_type": "gateway",
      "description": "网关离线",
      "status": 1,
      "status_text": "处理中",
      "handler_name": "张三",
      "created_at": "2026-03-20 09:30:00"
    }]
  }
}
```

**P03 POST /problems — 新增问题**

请求体：project_code / description / device_type / device_location / building / initial_cause / handler_id / attachments

响应：返回 id / problem_no / status / created_at

**P04 PUT /problems/:id — 更新问题**

请求体：status / action_taken / hours_spent

规则：
- 状态→处理中：action_taken 必填
- 状态→已完成：action_taken 和 hours_spent 均必填

**R02 POST /reports/inspection/export — 导出巡检报告Word**

请求体：project_code / start_date / end_date

响应：Word 文件流（.docx）

## 五、Docker 部署设计说明

### 5.1 容器架构

4容器架构：
| 容器 | 镜像 | 说明 | 端口 |
|------|------|------|------|
| frontend | nginx:alpine | 前端托管+Nginx反向代理 | 80 |
| backend | python:3.12-slim | FastAPI+Uvicorn | 8000 |
| mysql | mysql:8.0 | 数据库 | 3306 |
| file-storage | nginx:alpine | 文件存储服务 | 8080 |

### 5.2 docker-compose 结构

**开发环境：** 热重载开发模式，前端 Vite Dev Server + 后端 uvicorn --reload

**生产环境：** 前后端分离部署，后端 uvicorn 多 worker + Nginx 反向代理 + 静态文件分离

### 5.3 环境变量

.env 文件（勿提交）：
| 变量 | 说明 | 示例 |
|------|------|------|
| MYSQL_ROOT_PASSWORD | MySQL root密码 | xxx! |
| MYSQL_DATABASE | 数据库名 | problem_tracker |
| MYSQL_USER | 应用账号 | pt_user |
| MYSQL_PASSWORD | 应用密码 | xxx! |
| JWT_SECRET_KEY | JWT签名密钥（32字节hex） | secrets.token_hex(32) |
| JWT_ALGORITHM | JWT算法 | HS256 |
| ACCESS_TOKEN_EXPIRE_MINUTES | Token有效期（分） | 480 |
| UPLOAD_DIR | 上传目录 | /app/uploads |
| MAX_UPLOAD_SIZE_MB | 单文件大小上限 | 10 |
| ENVIRONMENT | 环境 | development / production |
| LOG_LEVEL | 日志级别 | DEBUG / INFO |
| CORS_ORIGINS | 允许的跨域源 | http://localhost:5173 |

### 5.4 部署步骤

**开发环境：**
```bash
cp .env.example .env   # 填入实际值
docker-compose -f docker-compose.dev.yml up -d
# 前端：http://localhost:5173
# 后端：http://localhost:8000
# API文档：http://localhost:8000/docs
```

**生产环境：**
```bash
cp .env.example .env.production  # 填入强密码
docker-compose -f docker-compose.prod.yml up -d --build
```

### 5.5 数据库初始化

MySQL容器首次启动时自动执行 docker/mysql/init.sql，完成：
- 建库（utf8mb4）
- 角色表初始化（超级管理员/项目管理员/普通用户）
- 管理员账号创建（username: admin / password: Admin@123456）
- 试点项目初始化（ABB P6 / BP）
- 设备类型字典初始化

### 5.6 备份与恢复

**备份：**
```bash
# 备份数据库
docker exec pt_mysql_prod mysqldump -u root -p problem_tracker > backup_$(date +%Y%m%d).sql

# 备份文件存储
docker cp pt_filestorage_prod:/usr/share/nginx/html/uploads ./uploads_backup_$(date +%Y%m%d)
```

**恢复：**
```bash
docker exec -i pt_mysql_prod mysql -u root -p problem_tracker < backup_YYYYMMDD.sql
```
